# PDF → XML (УПД/Акт ФНС 5.03)

Конвертер актов/УПД из PDF в XML формата ФНС (приказ № ЕД-7-26/970@). Структура совместима с Диадок, СБИС, Контур.

## Онлайн-версия

👉 **https://ВАШЛОГИН.github.io/pdf2xml/**

*(замените `ВАШЛОГИН` на ваш логин GitHub после публикации)*

Всё работает прямо в браузере: PDF не загружается ни на какой сервер, XML генерируется локально и скачивается файлом.

## Возможности

- Извлечение реквизитов продавца и покупателя (ИП / ЮЛ), ИНН/КПП, банк
- Таблица позиций с суммами
- Подписант, дата передачи, основания передачи
- Адрес: структурированный `АдрРФ` для Москвы/СПб/Севастополя, `АдрИнф` (текстом) для остальных
- Кодировка XML — windows-1251 (как у Диадока)
- Редактирование любых полей перед сохранением

## Как это работает

1. Загружаете PDF (перетаскиваете или кнопкой «Выбрать файл»)
2. Программа разбирает PDF через [pdf.js](https://mozilla.github.io/pdf.js/) и заполняет форму
3. Проверяете и при необходимости правите поля на трёх вкладках: шапка, позиции, подписант
4. Жмёте «Скачать XML» — файл сохраняется с именем исходного PDF

## Ограничения

- **Только текстовые PDF.** Сканы (картинки) не поддерживаются — нужен OCR, которого в приложении нет.
- **УПД со статусом 2** (передаточный документ без функции счёт-фактуры, т.е. без НДС). Для УПД со статусом 1 (с НДС) — добавьте ставки в коде.
- **XSD-валидация** не встроена. Итоговый XML стоит проверить в вашем ЭДО (Диадок/СБИС/Контур) — они укажут на несоответствия.

## Десктоп-версия (Python)

В корне репозитория лежит также Python-версия с GUI на tkinter (`main.py`, `parser.py`, `xml_builder.py`). Требует Python 3.10+ и `pdfplumber`:

```bash
pip install -r requirements.txt
python main.py
```

## Локальный запуск веб-версии

```bash
cd web
python -m http.server 8000
# откройте http://localhost:8000
```
